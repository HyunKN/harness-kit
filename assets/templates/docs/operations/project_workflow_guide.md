# Project Workflow Guide

## Goal

Keep implementation explainable and repeatable.

## Default loop

1. Align on the problem
2. Write a short spec first when scope or boundaries are still unclear
3. Create or update an issue doc
4. Break the work into small validated steps
5. Implement against the issue instead of relying on chat history
6. Update issue status, feature status, troubleshooting notes, decision records, review notes, and learning notes as appropriate

## Per-feature checklist

- scope has a written home
- issue exists
- implementation steps are clear
- implementation is validated
- docs are updated
- remaining risks are written down
