# Project Workflow Guide

## Goal

Keep implementation explainable and repeatable.

## Default loop

1. Align on the problem
2. Create or update an issue doc
3. Clarify scope and non-goals
4. Implement in small validated steps
5. Update issue status, feature status, and learning notes

## Per-feature checklist

- issue exists
- scope is clear
- implementation is validated
- docs are updated
- remaining risks are written down
