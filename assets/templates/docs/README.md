# Docs Hub

This folder is the entry point for project documentation.

## Recommended categories

- `architecture/` for structure and system design
- `product/` for scope and requirements
- `design/` for UX and interaction docs
- `reviews/` for analysis and review notes
- `status/` for implemented features and checklists
- `issues/` for real work backlog and issue tracking
- `records/` for important engineering decisions
- `learning/` for technical notes
- `playbooks/` for repeatable workflows
- `operations/` for project operating guides

## Recommended first reads

When this harness is installed into a repository, the preferred start order is:

1. `AGENTS.md`
2. `docs/README.md`
3. `docs/issues/current_backlog.md`
4. the issue doc for the current task
5. `docs/operations/agent_session_boot_protocol.md`
6. `docs/playbooks/repo_hygiene_gc_loop.md`
